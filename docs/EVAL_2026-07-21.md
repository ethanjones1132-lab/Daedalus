## Findings (ranked)

### F1 — Delegate is non-functional; every write lands via native fallback [CRITICAL]
All 4 `claude_cli/gemma4:e2b` attributions errored. `claude-delegate.ts:975` gates on
`permittedStockTools.has(stockToolName)`, where `permittedStockTools` holds **stock**
names (`Write`,`Edit`) parsed from `delegate.allowed_tools`. The delegate model emits
canonical-style names (`write_file`), which map to `delegate_write_file` and fail the
permit check → `delegate_tool_not_permitted`. P1's delegate never actually writes; the
native executor loop does. *Earlier "P1 success" live-fires were native fallback with a
claude_cli attribution on the failed delegate attempt — corrected here.*

### F2 — Conductor VRAM cascade times out the coordinator for minutes after a write turn [CRITICAL]
8GB GPU (~3.7GB used by desktop) holds the conductor `qwen3.5:4b` (3.7GB) **or** a
delegate model, not both. Any write turn evicts qwen; qwen's **cold reload is ~15s**
(`latency_ms=15164`), exceeding its own 15–20s coordinator budget, and `keep_warm`
only re-fires every 600s. Result: T2/T3 both timed out; 4/9 coordinator calls failed.
The warm-fallback fix (`e34fac7`) is **necessary but insufficient**: it correctly routes
to the resident gemma, but gemma's full-routing latency (12–48s) exceeds
`ROUTING_TIMEOUT_MS=10s`, so routeTurn still aborts → slow API coordinator. Manually
warming qwen between turns restored routing (proves the cause).

### F3 — `powershell` tool registered but unreachable [HIGH]
`shell-bundle.ts` registers `powershell` on win32, but it is absent from every
`modes.ts` `tools_filter`, so no stage can call it. The model wrapped PowerShell inside
`bash` instead. Feature 7's powershell path is dead as shipped.

### F4 — Permissive sandbox bypasses allowed-roots containment [HIGH/security]
T6 wrote to `C:\Users\ethan\Downloads\NonexistentDir99` (outside granted roots); the log
shows `Permissive mode: allowing access … (outside allowed roots)`. The P0.1
path-resolution containment is only enforced in **strict** mode; live config is
permissive, so message-mentioned or model-chosen paths are auto-created anywhere.

### F5 — Cross-turn session contamination [MEDIUM/reasoning]
T5's unfinished `Get-Date` sub-task resurfaced in the T6 **and** T7 synthesizer summaries
("PowerShell Get-Date … needs re-execution"), two turns later, on unrelated requests.
Session state carries stale sub-goals forward and pollutes synthesis.

### F6 — Severe latency [HIGH/perf]
Non-timeout turns ran 66–156s; synthesizer alone 17–50s per turn. Even successful turns
are far slower than interactive-usable.

### F7 — Reviewer `empty_completion` recurs [MEDIUM]
Reviewer produced empty output (T1); a known reasoning-model-burns-budget-in-thinking
failure that still leaks through on some stages.

### F8 — Initial tool call policy-blocked then retried [LOW/friction]
T1 delegate write and T4 first `bash` were both policy-rejected on first attempt, then a
re-scoped retry succeeded — wasted a stage turn each. Symptomatic of F1/scope handling.
